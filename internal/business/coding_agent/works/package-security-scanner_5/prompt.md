Create a package security scanner system that identifies vulnerable packages and provides remediation guidance. You need to create two main scripts:

1. **vulnerability_scanner.sh** - This script should:
   - Scan all installed Python packages using pip list
   - Cross-reference them against the vulnerability database at /app/data/vulnerability_db.json
   - Generate a security report named 'security_report.json' with the following structure:
     {
       "scan_timestamp": "ISO-8601 timestamp",
       "total_packages": number,
       "vulnerable_packages": number,
       "vulnerabilities": [
         {
           "package_name": "package name",
           "installed_version": "version",
           "vulnerability_id": "CVE or ID",
           "severity": "critical|high|medium|low",
           "affected_versions": "version range",
           "description": "vulnerability description",
           "remediation": "suggested fix"
         }
       ],
       "risk_score": number
     }

2. **policy_enforcer.sh** - This script should:
   - Read the security policy from /app/data/security_policy.json
   - Analyze the security report generated by the scanner
   - Generate remediation commands based on policy rules
   - Create an output file 'remediation_plan.txt' with specific pip commands to fix vulnerabilities
   - Exit with code 1 if critical vulnerabilities are found and policy enforcement is strict

The vulnerability scanner should calculate a risk score based on severity levels (critical=10, high=7, medium=4, low=1) and the policy enforcer should respect the maximum allowed risk score from the policy configuration.

Your solution should handle edge cases like missing packages or invalid version formats gracefully.
